# 📊 Product Performance Analysis Using MySQL

## 🔎 Overview
This project analyzes product-level performance using **MySQL** for data processing and **Power BI** for visualization.  
It demonstrates advanced SQL techniques, business insights, and interactive dashboards to evaluate product performance.

---

## 🎯 Project Objectives
- Analyze product performance using SQL
- Identify top and low-performing products
- Study category-wise and seasonal trends
- Apply advanced SQL concepts for deeper insights

---

## 🛠️ Tools Used
- **Database:** MySQL  
- **Visualization:** Microsoft Power BI  

---

## 📂 Datasets
- **Calendar** – 731 records  
- **Customers** – 5,000 records  
- **Products** – 200 records  
- **Sales** – 1,000,000 records  
- **Stores** – 100 records  

---

## 📈 Key Analysis
1. Total revenue by product  
2. Products below average revenue  
3. Revenue contribution analysis  
4. Top products by category  
5. Monthly category trends  

---

## 🧮 SQL Concepts Applied
- Common Table Expressions (CTEs)  
- Window Functions  
- `RANK()` and `DENSE_RANK()`  
- Aggregate Functions  
- Joins and Subqueries  

---

## 💡 Recommendations
- Promote top-selling products  
- Improve low-performing products  
- Plan stock using monthly trends  
- Focus on high-revenue categories  

---

## ⚠️ Challenges Faced
- Handling **1M+ records** efficiently  
- Managing complex joins  
- Writing advanced SQL queries  
- Performing time-based analysis  

---

## ✅ Conclusion
This project highlights the use of SQL for large-scale data analysis, extracting business insights, and presenting results through Power BI dashboards.  
It demonstrates strong analytical, technical, and visualization skills.
